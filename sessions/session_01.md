## ステップ 1: `/init` でプロジェクトコンテキストを自動生成

### 🎯 目的

ClaudeCode に `/init` コマンドを実行して、プロジェクト全体を自動解析させ、**CLAUDE.md** ファイルを生成します。このファイルが Claudeの「プロジェクト理解」の核になります。

### 💬 参加者の操作

```
1. ClaudeCode チャットを開く
   ↓
2. /init コマンドを実行
   ↓
3. Claudeが解析中...（数十秒待機）
   ↓
4. CLAUDE.md ファイルが生成される
   ↓
5. ファイルペインで CLAUDE.md を確認
```

### 📚 何が起きているのか

Claudeが **自動的に** 以下を実行：

- 📁 プロジェクトディレクトリ構造をスキャン
- 📦 `package.json` から技術スタックを抽出
- 🔧 開発コマンドを検出
- 🏗️ ディレクトリレイアウトからアーキテクチャを理解
- ✅ テストフレームワークやベストプラクティスを判定

### 💡 重要ポイント

**これが「ClaudeCode の違い」を体験する瞬間です。**

従来の Claude では「どこをどう見たらいい？」という質問から始まりますが、ClaudeCode は **自動的に全体を把握** した状態でサポートを始めます。

## ステップ 2: 生成された CLAUDE.md をざっと読む

### 🎯 目的

生成された CLAUDE.md ファイルで、このプロジェクトがどんなものなのかを理解します。**全部を読む必要はありません**。以下の 4 セクションだけで十分です。

### 💬 参加者の操作

```
1. エディタで CLAUDE.md を開く
   ↓
2. 以下の 4 セクションを読む：

   📋 プロジェクト概要
      → このアプリは何をするのか

   🛠️ 開発コマンド
      → よく使う npm コマンド

   🏗️ プロジェクトアーキテクチャ
      → ファイルの構成とデータの流れ

   📍 ルート一覧
      → 各ページはどこにあるか
```

### 📚 何を学ぶのか

- **Claudeが完全にプロジェクトを理解している状態** になったのを実感
- 今後の質問に対して、Claudeはプロジェクト固有の正確な回答ができる
- 例：「チケット作成機能はどこ？」と聞いたら、Claudeは `lib/actions/tickets.ts` や `lib/graphql/` を即座に知っている

### 💡 学習ポイント

- 🚀 **効率性の向上：** ファイルを手探りで探す時間がゼロに
- 🎯 **質問の精度向上：** 正しいコンテキストがあれば、正しい回答が返ってくる
- 📖 **ドキュメントの質：** AI が自動生成したドキュメントとは思えないほど実用的

## ステップ 3: Claudeに質問してコード検索を体験

### 🎯 目的

CLAUDE.md コンテキストを使って、Claudeが **いかに正確に** コード情報を提供できるか体験します。

### 💬 参加者の操作

```
1. チャットで以下の質問を入力：

   「チケット一覧ページはどこに実装されていますか？
    ファイルパスと主な実装内容を教えてください」

   ↓

2. Claudeが CLAUDE.md の情報を使って回答
   ↓

3. 回答を読んで、回答の詳しさと正確さを確認
```

### 📝 期待される回答例

```
チケット一覧ページは `/tickets` ルートで実装されています。

主なファイル：
- app/tickets/page.tsx（ページコンポーネント）
- components/tickets/TicketListContent.tsx（コンテンツ）
- lib/data/tickets.ts（Prismaクエリ）

フロー：
1. Server Component で lib/data/tickets.ts から getTickets() を実行
2. TicketListContent に表示データを渡す
3. チケット一覧をレンダリング
```

### 💡 学習ポイント

- **CLAUDE.md なし：** 「ファイルどこだろう...」と手探り
- **CLAUDE.md あり：** 即座に構造と実装パターンが返ってくる
- 🚀 **これが「コンテキスト付きの効率性」の実体験**

### 🎲 追加チャレンジ

他の質問も試してみましょう：

- 「Prisma のスキーマはどこにありますか？」
- 「チケット作成時のバリデーションルールは？」
- 「テストはどこに書かれていますか？」

各質問で、CLAUDE.md がいかに使われているか観察してください。
